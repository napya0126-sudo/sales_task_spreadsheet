# clasp設計アップデート要件定義書（スプレッドシート主導・一方通行版）

## 1. 背景・課題
- 現在は、Googleドキュメント内で見出しを作成し、そのヘッダリンクを手動でコピーしてスプレッドシートのH列（Doc URL）へ貼り付けている。
- この手動作業は工数が多く、貼り付けミスや更新漏れが発生しやすい。
- ドキュメント側を先に編集するのではなく、スプレッドシートに書いた「タスク名」をそのままドキュメントへ「見出し」として送り、リンクを自動取得したい。

## 2. 目的・ゴール
- スプレッドシートのカスタムメニューから実行することで、対象行の「タスク名」をドキュメント末尾に新規見出しとして追加する。
- 追加した見出しの「アンカー付きURL」を自動生成し、スプレッドシートのH列に書き戻す。
- 「スプレッドシートに入力 → ボタン実行 → リンク完了」の最短フローを実現する。

## 3. 対象範囲
### 3.1 対象システム
- Googleスプレッドシート（タスク管理用）
- Google Apps Script（clasp管理 / Google Docs API併用）
- Googleドキュメント（不具合再現シート / ID: `1ARHHXvoIgy5yS8Mi8PerplKcpGBkB59u5BOMbuYolmSPmzOw3s18UGdK`）

### 3.2 対象データ
- **タスク名（B列）**: ドキュメントに追加する見出しのタイトルとして使用。
- **Doc URL（H列）**: 生成された見出しリンクの格納先。

### 3.3 対象外
- ドキュメント内の既存見出しの検索および再利用（常に新規作成する）。
- スプレッドシートのタスク名自体のハイパーリンク化（H列への書き込みに限定）。
- 複数ドキュメントへの振り分け。

## 4. 前提条件
- H列に記録する値は、直接対象見出しへジャンプできる「アンカー付きURL（#heading=h.xxx）」とする。
- 見出しのスタイルは「見出し 3 (HEADING3)」をデフォルトとする。
- 実行ユーザーが対象ドキュメントの編集権限を持っていること。
- **Google Docs API** を Advanced Service として有効化すること。

## 5. 業務フロー（自動化後）
1. スプレッドシートの「タスク名（B列）」に内容を入力する。
2. 対象行のセルをどこでも良いので選択した状態にする。
3. カスタムメニュー「タスク管理 ＞ 🔗 Docに見出しを作成」を実行。
4. **（自動処理）**
   - 対象ドキュメントの末尾に、タスク名のテキストで新しい見出しを作成。
   - 作成した見出しの内部IDを取得し、アンカーURLを生成。
   - 生成したURLをスプレッドシートの対象行・H列に書き込む。
5. 完了通知が表示され、H列から直接ドキュメントの該当箇所へ飛べるようになる。

## 6. 機能要件
### 6.1 実行インターフェース
- スプレッドシート上部のカスタムメニュー「タスク管理」内に項目を追加する。
- 現在選択されているセルの「行」を対象とする。

### 6.2 タスク名取得
- B列（タスク名列）からテキストを取得する。
- タスク名が空欄の場合は「タスク名を入力してください」とエラー通知し、処理を中断する。

### 6.3 見出し作成（一方通行）
- 対象ドキュメントの最後に改行を加え、新しい「見出し 3」を追加する。
- 既存の見出しとの重複チェックは行わず、実行のたびに新しく作成する（運用上のシンプルさを優先）。

### 6.4 見出しリンク生成
- Google Docs API を使用して、新しく作成された段落（見出し）の `headingId` を取得する。
- ドキュメントのベースURLに `#heading=h.[headingId]` を付与してURLを完成させる。

### 6.5 H列書き戻し
- 生成したURLを対象行のH列（Doc URL）にセットする。
- 既にH列にURLが存在する場合、誤操作防止のため「上書きしてよいか」をダイアログで確認するか、あるいは既存値を保護する仕様とする（実装時に確定）。

## 7. 非機能要件
- **性能**: 1件の処理につき数秒程度で完了すること。
- **保守性**: ドキュメントIDや対象列番号をスクリプトの定数として管理し、変更を容易にする。
- **拡張性**: 複数行を選択して一括処理する機能への拡張を考慮した設計とする。

## 8. エラー・例外処理
- ドキュメントIDが不正、またはアクセス権限がない場合はエラーを表示する。
- ネットワークエラー等でAPI実行が失敗した場合は、その旨を通知する。

## 9. 受け入れ基準
- スプレッドシートから実行し、ドキュメントの最後に新しい見出しが作られていること。
- スプレッドシートのH列に入ったリンクをクリックした際、ドキュメント内の正しい（作成したばかりの）見出しへジャンプすること。
- タスク名に特殊文字が含まれていても正しく見出しが作成されること。
